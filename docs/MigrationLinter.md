# MigrationLinter

The `MigrationLinter` is a static analysis tool for SQL migration scripts, particularly those generated by Entity Framework Core migrations. It evaluates migration files for potential issues, risks, and anti-patterns, producing a structured report of findings that can be used to enforce quality gates in CI/CD pipelines or during local development.

## API

### `public MigrationLinter`

The primary entry point for the linter. This type encapsulates the linting process and provides methods to analyze migration files and retrieve results.

### `public static MigrationLinter CreateDefault()`

Creates a `MigrationLinter` instance configured with default rules and settings. This is the recommended way to obtain a linter instance unless custom rule sets or configuration are required.

- **Parameters**: None
- **Return Value**: A new `MigrationLinter` instance with default configuration.
- **Exceptions**: None

### `public LintReport Lint(IEnumerable<MigrationFile> migrations)`

Analyzes a collection of migration files and generates a lint report containing all findings, risks, and summary statistics.

- **Parameters**:
  - `migrations`: An enumerable of `MigrationFile` objects representing the migrations to analyze.
- **Return Value**: A `LintReport` containing the aggregated results of the linting process.
- **Exceptions**:
  - `ArgumentNullException`: Thrown if `migrations` is `null`.
  - `InvalidOperationException`: Thrown if the linter is in an invalid state (e.g., required rules are missing).

### `public IReadOnlyList<LintFinding> Findings { get; }`

Gets the collection of all findings discovered during the most recent linting operation. The list is immutable and reflects the state after the last call to `Lint`.

- **Return Value**: An `IReadOnlyList<LintFinding>` of all findings, or an empty list if no linting has been performed.
- **Exceptions**: None

### `public int MigrationsScanned { get; }`

Gets the number of migration files that were processed during the most recent linting operation. This value is zero if no linting has been performed.

- **Return Value**: The count of migration files scanned.
- **Exceptions**: None

### `public bool HasBlockers { get; }`

Determines whether any findings of severity `Blocker` were discovered during the most recent linting operation.

- **Return Value**: `true` if at least one `Blocker` severity finding exists; otherwise, `false`.
- **Exceptions**: None

### `public RiskLevel MaxRisk { get; }`

Gets the highest risk level among all findings discovered during the most recent linting operation. If no findings exist, returns `RiskLevel.None`.

- **Return Value**: The maximum `RiskLevel` encountered.
- **Exceptions**: None

### `public LintReport`

A structured report containing the aggregated results of a linting operation, including all findings, summary statistics, and risk assessments. Returned by the `Lint` method.

## Usage

### Basic Usage

```csharp
using var linter = MigrationLinter.CreateDefault();
var migrations = LoadMigrationFiles(); // Load migration files from disk
var report = linter.Lint(migrations);

Console.WriteLine($"Scanned {report.MigrationsScanned} migrations");
Console.WriteLine($"Max risk: {report.MaxRisk}");

if (report.HasBlockers)
{
    Console.WriteLine("Blocker findings detected!");
}
```

### Advanced Usage with Custom Rules

```csharp
using var linter = MigrationLinter.CreateDefault();
linter.AddRule(new CustomDestructiveOperationRule()); // Add a custom rule

var migrations = LoadMigrationFiles();
var report = linter.Lint(migrations);

foreach (var finding in report.Findings)
{
    Console.WriteLine($"{finding.RuleId}: {finding.Message}");
}
```

## Notes

- The `Lint` method is not thread-safe; concurrent calls on the same `MigrationLinter` instance will produce undefined behavior. If parallel linting is required, create separate instances per thread or synchronize access externally.
- The `Findings`, `MigrationsScanned`, `HasBlockers`, and `MaxRisk` properties reflect the state after the most recent call to `Lint`. If no linting has been performed, these properties return default values (empty list, zero, `false`, and `RiskLevel.None`, respectively).
- The linter does not modify the input `MigrationFile` objects during analysis. All findings are derived from static analysis of the migration content.
- Custom rules added via extension methods (e.g., `AddRule`) are not persisted across `CreateDefault` calls. Each new instance starts with the default rule set unless rules are added programmatically.
